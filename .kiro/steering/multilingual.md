# 日英バイリンガルドキュメント管理ルール

## 基本方針

- **デフォルト言語**: 日本語
- **英語版**: 日本語版と同期必須
- **コード内コメント**: 英語
- **コミットメッセージ**: 英語 (Conventional Commits)

## ドキュメント構造

```
docs/
├── ja/          # 日本語（プライマリ）
├── en/          # 英語（同期必須）
└── images/      # 共通画像（言語非依存）

integrations/<vendor>/docs/
├── ja/          # ベンダー固有ドキュメント（日本語）
└── en/          # ベンダー固有ドキュメント（英語）
```

## 同期ルール

1. **日本語を先に書く**: 日本語版を作成してから英語版を作成
2. **構造を一致させる**: 見出し構造、セクション順序は両言語で同一
3. **画像は共有**: `docs/images/` に配置し、両言語から参照
4. **コード例は同一**: コードブロックは言語間で同一内容
5. **更新時は両方更新**: 片方を更新したら必ず他方も更新

## README.md の言語スイッチャー

プロジェクトルートの README.md にはバイリンガルスイッチャーを配置:

```markdown
🌐 [日本語](docs/ja/getting-started.md) | [English](docs/en/getting-started.md)
```

## 翻訳品質ガイドライン

- 直訳ではなく、各言語で自然な表現を使用
- 技術用語は英語のまま使用（Lambda, CloudFormation, API 等）
- AWS サービス名は公式表記に従う
- 日本語固有の表現（「〜してください」等）は英語では適切に変換
