## 7. 前提条件

### 顧客が提供するもの
- [ ] FSx for ONTAP ファイルシステムを持つ AWS アカウント
- [ ] CloudFormation デプロイのための IAM 権限
- [ ] 対象 SVM で監査ログが有効（または有効化の許可）
- [ ] ベンダープラットフォームアカウント（または無料枠作成の意思）
- [ ] ネットワークアクセス: Lambda がベンダー API に到達可能（VPC の場合は NAT Gateway）

### パートナー/SA が提供するもの
- [ ] CloudFormation テンプレート（本リポジトリより）
- [ ] デプロイガイダンスとトラブルシューティング
- [ ] ダッシュボードとアラート設定
- [ ] 所見と推奨事項を含む PoC レポート

## 8. リスクと軽減策

| リスク | 発生可能性 | 影響度 | 軽減策 |
|------|-----------|--------|-----------|
| S3 AP ネットワークタイムアウト（VPC + Gateway EP のみ） | 中 | 高 | Lambda を VPC 外にデプロイ、または NAT Gateway を追加 |
| ベンダー API レート制限 | 低 | 中 | 指数バックオフを実装（Lambda コードに組み込み済み） |
| 監査ログフォーマット不一致 | 低 | 中 | 実ログの前にサンプルデータでテスト |
| 越境データ転送の懸念 | 中 | 高 | データレジデンシーマトリクスをレビュー、適切なリージョンを選択 |
| PoC コスト超過 | 低 | 低 | 無料枠ベンダーを使用、請求を毎日監視 |
