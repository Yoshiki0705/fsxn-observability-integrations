# ガバナンスとコンプライアンスに関する考慮事項

## 対象範囲

本リポジトリは、FSx for ONTAP のシグナル（監査ログ、EMS イベント、FPolicy ファイル操作）に対するオブザーバビリティパイプラインパターンを提供します。ワークロード固有のコンプライアンス制御、監査ポリシー、バックアップ、DR、アクセスガバナンスを**置き換えるものではありません**。

本パイプラインは**オブザーバビリティおよびエビデンス層**として機能し、既存の HA、バックアップ/リストア、DR メカニズムを補完するものです。

## 責任分界

| 領域 | お客様の責任 | AWS の責任 | パイプラインの範囲 |
|------|------------|-----------|-----------------|
| 監査ログ生成 | ONTAP 監査ポリシーの設定 | FSx for ONTAP サービス運用 | 読み取りと転送 |
| データ分類 | 監査ログの機密性分類 | 保存時暗号化 (SSE-FSx) | 転送のみ |
| シークレット管理 | API キー/トークンのローテーション | Secrets Manager サービス | 取得とキャッシュ |
| ネットワークセキュリティ | VPC、Security Groups、NACLs | 物理インフラストラクチャ | Lambda/Fargate の配置 |
| 送信先承認 | SaaS 送信先の承認 | N/A | 承認済みエンドポイントへの配信 |
| 保持ポリシー | 保持要件の定義 | ストレージ耐久性 | DLQ 保持のみ |
| 変更管理 | CloudFormation 変更のレビュー | サービス可用性 | テンプレートベースのデプロイ |
| インシデント対応 | エスカレーション手順の定義 | サービスヘルス通知 | 配信失敗時のアラート |

## 統制領域

### データ分類
- 監査ログにはファイルパス、ユーザー名、IP アドレス、操作詳細が含まれる可能性がある
- EMS イベントにはボリューム名、ノード識別子、アラート条件が含まれる可能性がある
- FPolicy イベントにはフルパスやクライアント IP を含むファイル操作の詳細が含まれる
- 適用される規制の下で、これらが PII または機密データに該当するかを判断すること

### シークレット管理とトークンローテーション
- すべてのベンダー API キー/トークンは AWS Secrets Manager に保存
- Lambda は実行コンテキストごとにトークンをキャッシュ（コールドスタート時に更新）
- ローテーション手順: Secrets Manager を更新 → 次回コールドスタートで新しい値を取得
- CloudFormation パラメータや環境変数に平文のシークレットを含めない

### IAM 最小権限
- Lambda 実行ロールは特定の S3 Access Point ARN にスコープ
- Secrets Manager アクセスは特定のシークレット ARN にスコープ
- SSM Parameter Store アクセスはチェックポイントパラメータパスにスコープ
- SQS 権限は特定の DLQ ARN にスコープ
- ワイルドカード (*) リソース権限は使用しない

### エビデンス保持
- CloudWatch Logs: 保持期間は設定可能（デフォルト 30 日）
- DLQ メッセージ: リプレイ調査のために保持
- SSM チェックポイント: 最後に処理した監査ファイルを追跡
- CloudFormation スタックイベント: デプロイの監査証跡

### リプレイと重複配信の取り扱い
- SSM チェックポイントにより処理済みファイルの再処理を防止
- DLQ が配信失敗を捕捉し、手動リプレイに対応
- 冪等性はバックエンドの重複排除に依存（パイプラインレベルでは保証しない）
- DynamoDB オブジェクト台帳（Level 3）により exactly-once 追跡を実現

### 変更管理
- すべてのインフラストラクチャは CloudFormation テンプレートとして定義
- テンプレート変更はプルリクエストレビューを実施
- CI がデプロイ前に cfn-lint でテンプレートを検証
- スタック更新イベントがデプロイの監査証跡を提供

### 運用オーナーシップとエスカレーション
- 定義すべき事項: DLQ 深度、チェックポイント経過時間、配信エラーの監視担当者
- 定義すべき事項: 配信失敗が継続した場合のエスカレーションパス
- 定義すべき事項: 新規 SaaS 送信先追加の承認プロセス
- 定義すべき事項: ベンダー API トークンのローテーションスケジュール

## 規制環境向けレビューチェックリスト

規制環境へのデプロイ前に、以下を確認してください:

- [ ] 監査ログがデータ分類ポリシーに基づき分類されている
- [ ] 越境ログ転送が許容される（SaaS 送信先が別リージョン/国にある場合）
- [ ] SaaS 送信先（Grafana Cloud、Datadog、Splunk 等）がセキュリティチームに承認されている
- [ ] 各ログ階層の保持・削除要件が定義されている
- [ ] リプレイ手順が運用チームおよびセキュリティチームに承認されている
- [ ] DLQ メッセージが運用エビデンスとしてレビューされている
- [ ] IAM ロールが最小権限の原則に従っている（セキュリティレビューで検証済み）
- [ ] シークレットローテーション手順が文書化・テスト済みである
- [ ] CloudFormation テンプレートがコンプライアンス観点でレビューされている（cfn-guard ルール）
- [ ] 監視・アラートが配信失敗、チェックポイント停滞、DLQ 深度をカバーしている

## 適用可能なフレームワーク

本パイプラインパターンは、以下のフレームワークにおけるエビデンス要件を支援できます:
- **AWS Well-Architected Framework** — 運用上の優秀性およびセキュリティの柱
- **SOC 2** — 監視およびログ記録の統制
- **ISO 27001** — A.12.4 ログ取得および監視
- **FISC**（金融情報システムセンター） — 監査証跡要件
- **ISMAP** — クラウドサービスのセキュリティ評価

> **注意**: フレームワークへの準拠には、本パイプラインを超えた包括的な評価が必要です。本ドキュメントは、パイプラインが統制目標にどのように貢献するかを示すものです。
